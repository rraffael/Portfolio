Relatório — Portfolio Next.js Starter

Gerado em 22/06/2026

Visão Geral

Portfólio pessoal de página única (one-page) construído com Next.js (Pages Router) e React 18. Existe exatamente uma rota (pages/index.jsx). A página é composta por seções empilhadas em ordem fixa — Home, About, Contact, Projects, Work e Footer — com navegação por scroll-snap, menu desktop, menu hambúrguer no mobile e um seletor de idioma (inglês / português).

O projeto está em estágio inicial de scaffold: a arquitetura está montada e funcional, mas o conteúdo de todas as seções ainda é placeholder proveniente dos arquivos de tradução.

Regras de Negócio Válidas

Como se trata de um portfólio (e não de uma aplicação de domínio), esta aba documenta as invariantes de código e decisões de arquitetura que governam o projeto hoje.

Estado global mínimo: apenas o locale

O único estado de aplicação é o idioma ativo ('en' | 'pt'), mantido em useState em index.jsx e passado para baixo. Não há persistência (sem localStorage nem i18n routing do Next): um refresh sempre volta para 'en'.

Fonte: pages/index.jsx:5, CLAUDE.md.

Convenção da prop t

PortfolioPage cria t = (path) => getMessage(locale, path) e repassa como prop. As seções não importam o sistema de locale — só chamam t('secao.chave'). Ao adicionar uma seção, é preciso registrá-la no array sections e na lista sectionRefs.current[N] na ordem correta.

Fonte: components/PortfolioPage.jsx:24, CLAUDE.md.

Resolução de tradução com cascata de fallback

getMessage resolve dot-paths contra locales/<locale>.json; se não achar, cai para inglês; se ainda assim faltar, devolve a própria string do path. Os dois JSONs devem manter estrutura de chaves idêntica.

Fonte: lib/locales.js:18-25.

Rastreio de seção ativa por scrollTop

handleScroll compara scrollTop com o offsetTop de cada seção para definir activeSection. Cliques na nav usam scrollIntoView e ativam isAutoScrolling por 550ms. O scroll-snap é feito em CSS.

Fonte: components/PortfolioPage.jsx:50-64, styles/globals.css.

Estilização 100% global

Todo o estilo vive em styles/globals.css (variáveis CSS, clamp() para tipografia fluida, breakpoints em 900px e 640px). Não há CSS Modules nem styled-components.

Fonte: pages/_app.jsx, styles/globals.css.

Como o Processamento é Executado

Não há linter, runner de testes nem config de TypeScript no projeto — npm run build é o único gate de correção.

Comandos disponíveis

npm install     # instala dependências
npm run dev     # dev server em http://localhost:3000
npm run build   # build de produção (único gate de correção)
npm start       # serve o build de produção

Fluxo de renderização (runtime)

  1. pages/_app.jsx carrega o CSS global e monta o componente da página.
  2. pages/index.jsx inicializa locale = 'en' e renderiza PortfolioPage.
  3. PortfolioPage constrói t, monta header + nav + container de scroll, e renderiza as 6 seções na ordem fixa (Home, About, Contact, Projects, Work, Footer).
  4. Cada seção chama t('...') para puxar textos do dicionário do idioma ativo.
  5. SectionContact dispara, na montagem, um fetch para http://localhost:8000 e exibe loading/erro/sucesso.
  6. O scroll é monitorado por handleScroll para destacar a seção ativa; cliques na nav usam scrollIntoView com guarda de 550ms.
Fontes: CLAUDE.md, package.json, components/PortfolioPage.jsx.

Principais Pontos em Aberto

Não há ROADMAP.md nem TODO.md no projeto. Os itens abaixo combinam observações da documentação com divergências entre doc/código encontradas na revisão.

Fetch hardcoded para localhost:8000

Bloqueio em prod Impacto: em produção o fetch sempre cai no estado de erro. Decisão pendente: trocar a URL fixa por process.env.NEXT_PUBLIC_API_URL (com guarda para quando não estiver definida), alinhando código e .env.example.

Fonte: components/SectionContact.jsx:13, .env.example.

isAutoScrolling é estado morto

Bug Impacto: a flag é definida mas nunca lida por handleScroll; o "guard" descrito no CLAUDE.md não guarda nada — durante o scroll suave do clique, activeSection continua sendo sobrescrito. Decisão pendente: usar a flag (early-return em handleScroll) ou removê-la.

Fonte: components/PortfolioPage.jsx:18,38,46,50.

Campo component do array sections não utilizado

Refactor Impacto: o array define component: HomeSection etc., mas o JSX renderiza cada seção manualmente — duas fontes de verdade e risco de desalinhar o índice do ref. Decisão pendente: mapear via section.component (DRY) ou remover o campo.

Fonte: components/PortfolioPage.jsx:26-32,167-213.

Conteúdo todo placeholder

Conteúdo Impacto: Home, About, Projects, Work e Footer têm texto genérico; o portfólio ainda não é entregável. Decisão pendente: preencher com conteúdo real e revisar mistura EN/PT nos JSONs (ex.: about.title = "About Me" também no pt.json).

Fonte: locales/en.json, locales/pt.json.

Sem SEO / metadata e sem persistência de idioma

Melhoria Impacto: não há next/head (sem <title>, meta description ou favicon); e o idioma reseta no refresh. Decisão pendente: adicionar <Head> com metadados e persistir o locale (ex.: localStorage). A bandeira PT usa 🇵🇹 enquanto o conteúdo é PT-BR.

Fonte: revisão de código; components/PortfolioPage.jsx:12.
